# Report Bugs

You can report bugs at the [Issue List](https://code.google.com/p/fcitx/issues/list), but please read the precautions first, or you can also check the [FAQ](../Users/FAQ).

**The Issue List is not a forum, please don't ask questions there!**

## Input method cannot be used
**Please** read the [FAQ](../Users/FAQ) first. Often the problem you encounter is not a real bug.

## Bugs related to crashes
Please paste the content of your **~/.config/fcitx/log/crash.log** file first. And please confirm if you can still reproduce the crash. If you can reproduce it, try removing your ~/.config/fcitx/ directory and other related data, then try again to see if it still crashes. If you cannot reproduce it, please also paste your ~/.config/fcitx.

## Feature Requests
Some people may have some random ideas, but as developers, we may think it's not useful, or it may bring more confusion than useful features. You can try to [become a developer](../Developers/DeveloperFAQ) yourself. Please think about it before you actually make the request.

By the way, the fcitx and fcitx-dev mailing lists also welcome discussions. You can let developers verify your ideas and then submit them to the issue list to ensure they won't be forgotten.
