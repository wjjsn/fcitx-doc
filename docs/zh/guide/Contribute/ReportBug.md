# 报告Bug

您可以到[问题列表](https://code.google.com/p/fcitx/issues/list)这里报告漏洞, 不过还请您先阅读注意事项，或者您也可以先看看[FAQ](../Users/FAQ)。

**问题列表不是论坛，请不要在那里提问！**

## Input method cannot be used
**请** 先阅读 [FAQ](../Users/FAQ)。往往您遇到不是真正的bug。

## 崩溃相关的漏洞
请先贴上您的 **~/.config/fcitx/log/crash.log**文件内容。 并且请确认你是否还能重新复现崩溃。如果您能复现，请尝试移除您的 ~/.config/fcitx/目录, 以及与之相关的其他数据，之后再尝试是否还会崩溃, 如果不能复现，也请贴上您的~/.config/fcitx。

## 请求功能
有些人可能会有一些随意的想法，但作为开发人员，我们可能认为它没有用，或者它可能比有用的功能带来更多的混乱。 您可以自己尝试 [成为开发者](../Developers/DeveloperFAQ)。请在您真正提出要求之前先想一想。

顺便说一句，fcitx 和 fcitx-dev 邮件列表也欢迎讨论，您可以让开发人员验证您的想法，然后将其提交到问题列表，以确保不会被遗忘。
